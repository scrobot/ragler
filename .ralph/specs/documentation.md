## Quality Requirements

### Every page must include:
- Clear title and description
- Table of contents (auto-generated by Docusaurus)
- Code examples (with syntax highlighting)
- Screenshots where relevant (UI pages)
- Links to related pages

### Getting Started pages must:
- Work end-to-end when followed step by step
- Include expected output for each command
- Note prerequisites clearly at the top
- Handle error cases with troubleshooting tips

### API Reference pages must:
- Match actual Swagger/OpenAPI output
- Include request body schemas (with types)
- Include response body examples
- Document all headers (X-User-ID, Content-Type)
- Show curl examples for every endpoint

### Architecture pages must:
- Include Mermaid diagrams
- Explain design decisions (not just describe structure)
- Reference relevant source files

## Documentation Style Guide
- Use second person ("You can create a collection...")
- Use present tense for descriptions
- Use imperative for instructions ("Run the following command...")
- Keep paragraphs short (3-4 sentences max)
- Use tables for options/configs
- Use admonitions (:::tip, :::warning, :::info) for callouts
